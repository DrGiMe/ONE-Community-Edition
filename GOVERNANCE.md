# Gobernanza inicial de ONE

## Objetivo

Crear un proyecto abierto, entendible y modificable por la comunidad.

## Decisiones iniciales

- El proyecto se mantiene simple durante la etapa 0.x.
- Se prioriza facilidad de uso sobre potencia avanzada.
- La interfaz grafica es parte central del proyecto.
- La CLI existe para desarrolladores y automatizacion.

## Roles comunitarios sugeridos

- Usuarios: prueban ONE y reportan problemas.
- Documentadores: mejoran guias y ejemplos.
- Disenadores: mejoran experiencia visual.
- Desarrolladores: corrigen errores y agregan capacidades.
- Mantenedores: revisan Pull Requests y cuidan coherencia.

## Criterio rector

Una funcion nueva debe poder explicarse a una persona sin formacion tecnica en menos de un minuto.
