# Contributing to ONE

ONE Community Edition is open to users, designers, documenters, and developers.

## Ways to contribute

- Report bugs.
- Suggest templates.
- Improve documentation.
- Improve ONE Studio.
- Improve the `.one` language.
- Add tests.

## Basic workflow

1. Fork the repository.
2. Create a branch.
3. Make a small change.
4. Run tests.
5. Open a Pull Request.

## Design rule

Every change should make ONE easier, safer, or more useful for people who do not know how to program.
